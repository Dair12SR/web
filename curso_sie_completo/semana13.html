Semana 13: Ciclo de vida del desarrollo de sistemas y metodologías asociadas

Objetivos

  • Conocer las fases del ciclo de vida del desarrollo de sistemas (SDLC).
  • Entender las principales metodologías de desarrollo de software (ágiles, cascada, etc.).
  • Comparar las ventajas y desventajas de diferentes metodologías.

Contenidos

  • Ciclo de Vida del Desarrollo de Sistemas (SDLC): El SDLC define los pasos necesarios para desarrollar un sistema. Fases clave:
    • Planificación: Objetivos y alcance del proyecto.
    • Análisis de Requisitos: Necesidades y funcionalidades.
    • Diseño: Arquitectura y especificaciones técnicas.
    • Desarrollo: Codificación del sistema.
    • Pruebas: Verificación funcional.
    • Implementación: Despliegue del sistema.
    • Mantenimiento: Soporte y mejoras continuas.
  • Metodologías de Desarrollo de Software: Enfoques variados con distintos niveles de flexibilidad:
    • Waterfall: Lineal y secuencial. Ideal para proyectos bien definidos.
    • Ágiles: Iterativas y adaptativas. Scrum, Kanban, XP, etc.
    • DevOps: Combina desarrollo y operaciones para mayor eficiencia.
  • Herramientas de Desarrollo: Facilitadores clave para equipos modernos:
    • IDEs: Visual Studio Code, IntelliJ IDEA, Eclipse.
    • Control de Versiones: Git, GitHub, GitLab.
    • Automatización de Pruebas: Selenium, JUnit, Postman.

Actividades

  • Comparación de Metodologías: Tabla entre Waterfall y Scrum. Incluir ventajas, desventajas y escenarios.
  • Diagrama de Fases del SDLC: Crear un gráfico con descripciones breves de cada etapa.
